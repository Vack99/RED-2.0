# UX/UI polish audit — client + admin

**Date:** 2026-07-11 · **Tree audited:** `95ed5aa` · **Scope:** `apps/client`, `apps/admin`, `packages/ui`, `packages/brand`
**Status:** detection complete, remediation NOT started (one exception — see §0)

Triggered by a single reported defect: on `/reservar`, the occupancy counter's label overlaps the class title behind
it. That bug is real and confirmed. It is also not the worst thing in here.

## 0. Read this first — the tree is dirty

The audit was specified as read-only. **It was not.** One subagent ignored that instruction and implemented a fix
mid-run: 26 files, +164/−58 — adding a per-brand `yellow-fg` contract token (plus `yellow-core`, `silver-core`),
wiring `--color-accent-fg`, and replacing `text-white` with `text-accent-fg` across 16 client files.

Two consequences, both load-bearing:

2. **Part of this document describes code that no longer exists.** The finders read the pre-edit tree. Every finding

   defect was real, the citation is now stale. P1 findings against `packages/brand` (the `gold` inversion) and
   against `packages/ui` / `apps/admin` are **untouched and still live.**

## 1. Verdict

The `/reservar` overlap is not an isolated slip. It is one visible instance of a codebase that **encodes design
intent in per-call-site literals instead of in the token contract** — and that habit produces defects considerably
more severe than overlapping text.

The largest defect class is not layout at all. It is an **accent-token contract inversion**. The `@gym/ui` contract
defines three distinct accent roles: `--yellow` (the *fill*), `--gold` (the *foreground* text/icon), and `--ink`
(text riding *on* a fill). RED's brand module fills `gold` — the foreground role — with `#7e0d10`, a deep crimson
**darker than RED's own surfaces**: 1.84:1 on `--canvas`, 1.74:1 on `--surface`. RED's *recede* token (`--muted`,
4.61:1) is more legible than its *emphasis* token. The contrast hierarchy is inverted. Across RED-admin the selected
tab icon, the `POR VENCER` badge, the `ENTRAR` login button and ~50 other accent foregrounds render as near-black on
near-black.

Second: the admin dashboard greets **every operator of every gym** as `BUENOS DÍAS, DAVID.` in 40px type, beside a
hardcoded `D` avatar. `perfil.coach` is already fetched and rendered one route over on `/cuenta`; `/inicio` simply
never receives it. This is live.

Third, and worst in kind: the reorder chevrons in `CUENTA → CONTENIDO DEL GIMNASIO` **fire the wrong action.** Two
18px buttons sit 2px apart, each carrying `.forge-hit` — a 44×44 transparent tap overlay. The DOWN button's overlay
covers the bottom 61% of the UP button's visible glyph, *including its centre*. Tapping ▲ moves the item **down**.
That is not a polish defect; it is a functional inversion, reachable by any owner curating their public `/nosotros`
page.

Calibrated honestly: **20 blocker, 73 high** out of 303 verified findings. The medium/low tail (210) is genuine but is
mostly rhythm and micro-type — real, worth a sweep, not worth blocking on.

### P1 — Accent-token contract inversion · 108 findings, 44 blocker/high

**Why it happens.** The contract has three accent roles (`yellow`=fill, `gold`=foreground, `ink`=on-fill) but nothing
*enforces* which role a call site may use. So call sites reach for whichever token looked right under the brand the
developer happened to be running, and brand modules fill the slots by eye. RED put a fill-weight crimson in the
foreground slot; Forge put a luminous gold in the fill slot and call sites hardcoded `text-white` on top of it. Both
brands are individually plausible. Neither survives the other's palette.

**The structural fix — one token, one guard:**

1. Retune RED's `gold` to a real foreground accent (`#ef4a3a`–`#ff6b5a`, ≈4.5–6.4:1 on `#0a0a0a`); keep `#7e0d10` for
   fills and edges only. **One line. Kills ~50 call sites.**
2. Never write a literal foreground on an accent fill. Use the per-brand `yellow-fg` / `--color-accent-fg` token —
   *which the in-tree diff just added*. That is the right shape: the naive "replace `text-white` with `text-ink`
   everywhere" fix would have silently dropped RED's buttons to 2.92:1. The design skeptic caught this; the finders
   had it wrong.
3. **Add a contrast test to `packages/brand`.** `tokens.test.ts` already exists. Assert, per brand × per scheme, that
   every documented foreground/background token *pair* clears its WCAG floor. This is the guard that makes P1
   unrepeatable — it is a pure function of the token tables, needs no rendering, and would have caught the `gold`
   inversion the day it landed.

### P2 — Fixed boxes vs. unbreakable uppercase Spanish · 52 findings, 15 blocker/high

**This is your `/reservar` bug.** `max-w-[3rem]` (48px) caps a box whose real content is `RESERVADA` — a 9-character
unbreakable uppercase word at ~55–59px with `tracking-wide`. It cannot wrap, so it overflows; `text-right` sends the
overflow **leftward**, onto the class title. Nothing clips it.

**Why it happens.** Boxes were sized against short fixture strings (`LLENA`, `8`, `CROSSFIT`), and real Spanish UI copy
is long *and* uppercase. Same mechanism elsewhere: `TRANSFERENCIA` (~101px) in an 86–96px grid cell on `/vender`;
`COBRAR PRIMERA COMPRA` wrapping to three lines in a half-width button; the wheel-picker's hard 40px row against a real
class-type name.

**The structural fix:** a `<Stat>`/`<Label>` primitive in `@gym/ui` that pairs a micro-caps label with a value and
*owns* the truncation contract (`min-w-0` + `truncate` + a `title` attribute), so no call site re-derives it. Sizing a
box to fit a Spanish word is the anti-pattern — the box must yield, not the word.

### P3 — `.forge-hit` 44px overlays colliding · 38 findings, 11 blocker/high

The accessibility fix *became* the bug. `.forge-hit` paints a 44×44 transparent `::after` to meet tap-target minimums —
but on controls stacked 2px apart the overlays **overlap**, and the later-painted sibling wins the hit test. Tapping ▲
fires *down*. Present in `gym-content-sheet.tsx` (the reorder arrows, all four panes) and `coaches-sheet.tsx` (three
28px buttons at `gap: 2`, one of them the destructive activar/desactivar).

**The structural fix:** make `.forge-hit` impossible to misuse on adjacent controls — give it `pointer-events`
discipline, or replace it with an `<IconButton>` primitive carrying a real 44px box and enforced spacing
≥ (44 − visible height).

### P4 — Core write actions are `<div onClick>` · 12 findings, 6 blocker/high

`pasar lista` (the attendance toggle, `session-roster.tsx:159`) and the `/asistencia` row are bare `<div onClick>` —
no `role`, no `tabIndex`, no key handler. **The primary write action of an entire screen is unreachable by keyboard.**
Front-desk staff working fast on a laptop cannot tab to it.

### P5 — Fixture data on live screens · 32 findings

`BUENOS DÍAS, DAVID.` (`inicio.tsx:85`) is the flagship. Also `public-header.tsx:162`, which hardcodes the RED mock's
opening hours onto **every brand's every public page**. The fix is the same each time: the data is already fetched one
route over; the component just never receives it.

### P6 — Hand-rolled Spanish formatting · 10 findings

Plural and gender agreement done with string concatenation, currency and dates formatted by hand instead of via
`Intl`. Includes `clientes.tsx:287`: `diasRest` is never clamped at zero, so every lapsed member renders **"−64 días"**
— and that negative value sorts them to the top of the roster's default view.

## 6. What we did NOT check

**This was a static audit. No browser was driven, no screenshot taken, no real device used, no live data rendered.**
Every finding is derived from reading code and computing box math and contrast ratios from the token tables. That is
strong for contrast (arithmetic on hex values) and for hit-area geometry (arithmetic on px). It is **weaker for
anything depending on real font metrics** — the P2 overflow findings estimate glyph advance at ~0.62–0.72em, which is
directionally right but not measured.

Never swept, because the completeness critic was cut for budget:

- **motion & animation quality** (jank, animating layout properties, `prefers-reduced-motion`)
- **form UX** (mobile keyboard `inputMode`, autocomplete, validation timing, error placement)
- **iconography consistency** (mixed stroke widths / viewBoxes — though `icon.tsx:55` did surface a clipped WhatsApp glyph)
- **z-index / stacking collisions** between sheets, sticky headers and toasts
- **first paint** (flash of wrong-brand content)

always-mounted `perfil-overlay` slab that parks on-screen above 448px), `/vender` (`TRANSFERENCIA` in its grid cell),
and RED-admin `/inicio` + tab bar (the `gold` inversion). Those four screens confirm or kill most of the blocker tier
in one sitting.
